# presentations_and_materials

This repository serves as a centralized home for presentation materials
created and shared across the organization, including conference talks,
internal presentations, and supporting assets.

## 📁 Structure

Each conference, event, or standalone presentation should have its own
directory at the root of the repository.

    presentations_and_materials/
    │
    ├── conference_name_2026/
    │   ├── presentation_title/
    │   │   ├── slides.pptx
    │   │   ├── notes.md
    │   │   ├── assets/
    │   │   └── README.md
    │   │
    │   └── another_presentation/
    │
    └── internal_event_2026/

## 🧭 Guidelines

-   **One directory per presentation or talk**
    -   Use clear, descriptive naming:
        `event_name_year/presentation_title`
-   **Include source files whenever possible**
    -   (e.g., PowerPoint, Keynote, Google Slides export, Quarto, etc.)
-   **Add a local README for each presentation**
    -   Include context, audience, and key talking points
-   **Store supporting materials in an `assets/` folder**
    -   Images, datasets, diagrams, etc.

## 📌 Recommended Contents per Presentation

-   `slides.*` --- Final presentation deck
-   `notes.md` --- Speaker notes or outline
-   `assets/` --- Supporting files
-   `README.md` --- Overview of the presentation

## 🎯 Purpose

-   Preserve institutional knowledge from conferences and events\
-   Enable reuse and iteration on existing materials\
-   Provide visibility into what the organization is presenting
    externally

## 🤝 Contributing

1.  Create a new directory following the structure above\
2.  Add all relevant materials\
3.  Include a short README for context\
4.  Submit a pull request
